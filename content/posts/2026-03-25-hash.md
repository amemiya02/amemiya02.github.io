# 哈希

## [1. 两数之和 - Easy](https://leetcode.cn/problems/two-sum/)

### 题目回顾

> 给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值 `target`** 的那 **两个** 整数，并返回它们的数组下标。
>
> 你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。
>
> 你可以按任意顺序返回答案。

**示例 1：**

**输入：** `nums = [2,7,11,15]`, `target = 9`
**输出：** `[0,1]`

---

### 核心思路：用空间换时间

使用哈希表（字典）记录已经遍历过的元素，实现 $O(1)$ 查找。

---

### 代码实现（Python）

```python
def twoSum(nums, target):
    mp = {}
    for i, num in enumerate(nums):
        if target - num in mp:
            return [mp[target - num], i]
        mp[num] = i
    return [-1, -1]
````

---

### 复杂度分析

* 时间复杂度: $O(n)$
* 空间复杂度: $O(n)$

## [49. 字母异位词分组 - Mid](https://leetcode.cn/problems/group-anagrams/)

### 题目回顾

> 给你一个字符串数组，请你将 **字母异位词** 组合在一起。

---

### 核心思路：构造唯一 key

用字符频率作为 key（或排序字符串）

---

### 代码实现（Python）

👉 推荐写法（计数法，最优）

```python
from collections import defaultdict

def groupAnagrams(strs):
    mp = defaultdict(list)

    for s in strs:
        cnt = [0] * 26
        for c in s:
            cnt[ord(c) - ord('a')] += 1
        mp[tuple(cnt)].append(s)

    return list(mp.values())
```

👉 简洁写法（排序法）

```python
from collections import defaultdict

def groupAnagrams(strs):
    mp = defaultdict(list)

    for s in strs:
        key = "".join(sorted(s))
        mp[key].append(s)

    return list(mp.values())
```

---

### 复杂度分析

设 N 为字符串数量，K 为字符串长度：

* 时间复杂度: $O(N \cdot K)$（计数）或 $O(N \cdot K \log K)$（排序）
* 空间复杂度: $O(N \cdot K)$

## [128. 最长连续序列 - Mid](https://leetcode.cn/problems/longest-consecutive-sequence/)

### 题目回顾

> 给定一个未排序的整数数组 `nums`，找出最长连续序列长度，要求 $O(n)$。

---

### 核心思路：哈希集合 + 起点剪枝

关键优化：

👉 只从“序列起点”开始计算

判断条件：

```
num - 1 不存在 → 才是起点
```

---

### 代码实现（Python）

```python
def longestConsecutive(nums):
    s = set(nums)
    ans = 0

    for num in s:
        # 只从起点开始
        if num - 1 not in s:
            cur = num
            length = 1

            while cur + 1 in s:
                cur += 1
                length += 1

            ans = max(ans, length)

    return ans
```

---

### 复杂度分析

* 时间复杂度: $O(n)$
  👉 每个元素最多被访问 2 次
* 空间复杂度: $O(n)$
